# ONNODE TV 연결 설명서

TV를 **연습실 큰 화면**으로 쓰는 방법입니다.  
**TV 연결은 선택**이며, 연결하지 않아도 스마트폰·노트북만으로 연습할 수 있습니다.

---

## 1. 한눈에 보기

| 기기 | 하는 일 |
|------|---------|
| **TV** | 안무 영상, AI 코치, 내 모습(큰 화면), 실시간 피드백 |
| **스마트폰 / 노트북** | 카메라(댄스) 또는 마이크(보컬), 시작·종료 버튼 |

---

## 2. 연결 순서 (3단계)

### 1단계 — 모바일에서 TV 연결 열기

1. ONNODE 로그인
2. 왼쪽 메뉴 **💪 트레이닝** 선택
3. 기획사·댄스/보컬 선택 → **트레이닝 시작**
4. 화면 **우측 상단 `📺 TV 연결`** 클릭

→ 자동으로 **TV에 입력할 주소**와 **6자리 연결 코드**가 표시됩니다.

### 2단계 — TV에서 주소 열기（가장 중요）

TV 리모컨으로 **인터넷(브라우저)** 앱을 실행한 뒤, **모바일에 보이는 주소를 TV 키보드로 직접 입력**합니다.

> **QR 코드는 뭔가요?**  
> 폰/노트북 화면에 보이는 QR은 **위 주소와 같은 정보**를 담은 그림입니다.  
> **TV에 폰 화면을 비춰서 QR을 찍으라는 뜻이 아닙니다.**  
> 대부분의 TV는 QR 스캔이 안 되므로, **주소를 직접 입력**하는 것이 정석입니다.

**예시 주소 형식**

```
https://your-domain.com/tv?code=482910
```

배포 후에는 `https://onnode.ai/tv?code=482910` 형태가 됩니다.

| TV 종류 | 브라우저 여는 방법 |
|---------|-------------------|
| **삼성 Smart TV** | 홈 → **Internet** 앱 → 상단 주소창 탭 |
| **LG webOS** | 홈 → **웹브라우저** → 주소 입력 |
| **Google TV / Chromecast** | 앱 → **Chrome** → 주소 입력 |
| **Apple TV** | **Safari** → 주소 입력 |
| **노트북 + HDMI** | TV에 HDMI 연결 → 브라우저에서 `/tv` 접속 → **F11** 전체화면 |

（참고）일부 삼성·LG TV는 브라우저에 QR 스캔 기능이 있어, 그때만 폰 화면의 QR을 TV 카메라로 비출 수 있습니다.

### 3단계 — 모바일에서 카메라/마이크 켜기

1. TV 연결 모달을 **닫거나** 「연습 시작」 클릭
2. 트레이닝 화면에서 **「시작」** 또는 **「카메라 켜기」** (댄스) / **「마이크 켜기」** (보컬)
3. 브라우저 권한 창 → **허용**
4. 상단에 **STUDIO LIVE** 표시되면 연결 완료

## 3. TV 화면 구성 (댄스)

```
┌─────────────────────────┬─────────────────────────┐
│  왼쪽                    │  오른쪽                  │
│  · AI 코치 (페르소나)     │  · 내 카메라 실시간       │
│  · 유튜브 안무 시범 영상  │  · 동작 스켈레톤         │
│  · 점수 · 박자 정확도     │                         │
├─────────────────────────┴─────────────────────────┤
│  실시간 피드백 (예: "오른팔을 10도 더 올리세요")      │
└───────────────────────────────────────────────────┘
```

**안무 영상**은 모바일에서 유튜브 URL을 **불러온 뒤** TV 왼쪽에 표시됩니다.  
(TV 연결 전에 불러 두면 편합니다.)

---

## 4. 연결 방법 2가지

### 방법 A — 모바일이 코드 생성 (일반적)

1. 모바일: `📺 TV 연결` 클릭 → TV에 넣을 **주소** 표시
2. TV: 브라우저 주소창에 `https://도메인/tv?code=6자리숫자` **직접 입력**

### 방법 B — TV가 코드 생성

1. TV: `https://도메인/tv` 접속 → 화면에 **6자리 코드** 표시
2. 모바일: `📺 TV 연결` → 「TV에 다른 코드가 보이면 입력」→ 6자리 입력 → **연결**

## 5. 자주 묻는 질문

### 버튼을 눌렀는데 아무 일도 안 일어나요

다음을 확인하세요.

1. **Firebase 설정** — `.env`에 `VITE_FIREBASE_*` 값이 있는지
2. **익명 로그인** — Firebase Console → Authentication → 익명 사용 설정
3. **Firestore 규칙** — `firestore.rules.example` 참고
4. 모달에 **연결 코드·QR**이 보이는지 (없으면 「다시 시도」 클릭)
5. 코드가 보인 뒤 **카메라/마이크를 켰는지** (켜야 TV에 영상 전송)

### TV 없이 연습할 수 있나요?

**네.** `📺 TV 연결`을 누르지 않으면 폰/노트북 한 화면에서 영상+카메라로 연습합니다.

### iPhone / 갤럭시 / 노트북 모두 되나요?

**네.** 카메라·마이크가 있는 기기면 됩니다.

### AirPlay·스마트뷰는요?

앱 내 **WebRTC 연결**이 가장 정확합니다 (왼쪽 안무 / 오른쪽 내 모습 분할).  
AirPlay·HDMI 미러링은 화면 전체를 TV에 띄우는 **보조 방법**입니다.

## 6. 관리자 설정 (최초 1회)

### Firebase Authentication

1. [Firebase Console](https://console.firebase.google.com) → Authentication
2. Sign-in method → **익명(Anonymous)** → **사용 설정**

### Firestore 보안 규칙

프로젝트 루트 `firestore.rules.example` 파일을 참고해 아래 경로를 인증 사용자에게 허용합니다.

- `artifacts/{appId}/public/data/tvSessions/{code}`
- `artifacts/{appId}/public/data/sessions/{code}/webrtc/...`

### 환경 변수 (`.env`)

```
VITE_FIREBASE_API_KEY=...
VITE_FIREBASE_AUTH_DOMAIN=...
VITE_FIREBASE_PROJECT_ID=...
VITE_FIREBASE_APP_ID=...
```

회사·학교 Wi-Fi 등에서 연결이 잘 끊기면 TURN 서버 추가:

```
VITE_WEBRTC_TURN_URLS=turn:...
VITE_WEBRTC_TURN_USERNAME=...
VITE_WEBRTC_TURN_CREDENTIAL=...
```

---

## 7. 같은 PC에서 테스트하는 방법

1. 모바일(또는 브라우저)에서 `📺 TV 연결` 클릭
2. 모달의 **「TV 화면 미리보기」** 클릭 → 새 탭에 TV 화면 열림
3. 원래 탭에서 **카메라 켜기** → 두 탭이 연결되는지 확인

---

## 8. 기술 구조 (참고)

- **Firestore** — 점수, 피드백, 스켈레톤, 영상 URL 동기화
- **WebRTC** — 모바일 카메라 → TV 실시간 전송
- **MediaPipe** — 모바일에서 포즈 분석 → TV에 스켈레톤 표시
